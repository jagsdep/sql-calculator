Introduction

In this exercise, you’ll practice using intermediate and advanced SQL syntax, including aggregate functions, string manipulation, JOINs, and more.

Part 1: SQLZoo Practice

Head over to SQLZOO and complete the following sections to warm up your SQL skills for today’s exercise.

  • Basic SQL

  • SUM and COUNT

  • JOIN

Part 2: Database Setup

**Note: This exercise assumes you have PostgreSQL installed on your machine. If you do NOT have PostgreSQL installed, either attempt the installation, or get in the queue to ask for help getting it installed.

  1. Download the files for this exercise on Frodo.

  2. On the command line, go into the directory where the exercise files exist.

  3. On the command line, run the following command to create a new database.

$ createdb projects

  1. Next, run the SQL file so that it creates rows/tables in the new database you created. You should see some SQL output after you run this.

$ psql projects < database.sql

  1. Access the interactive database console.

$ psql projects

  1. Run a practice query. Make sure you see the proper results.

projects=# SELECT * FROM projects;

If you are not able to complete any of the above steps, ask for help in the queue.

Part 3: Calculator & More

Begin a file, projects.sql where you will add each query that you write. Please label your queries using SQL comments, which begin with the characters --. Make sure to include a version of the result of each query as a comment below the query.

For example, if question one asked you to write a query to find the id of the project News Aggregator, here is what you should add to the file.

-- Problem 1

SELECT id FROM projects WHERE title = 'News Aggregator';
-- Result: 2

Basic Info

  1. What is the average grade for the project called News Aggregator? Be sure to use the AVG aggregate function in your query.

  2. What is the sum of all points that the entire class received on the project called Recipe Storage? Be sure to use the SUM aggregate function in your query.

  3. How many total projects are there? Be sure to use the COUNT aggregate function in your query.

  4. What is the maximum grade that students received on the project called News Aggregator? Be sure to use the MAX function in your query.

  5. What is the minimum project that any student received on ANY project? Be sure to use the MIN function in your query.

JOINs

  1. Produce a result set that shows each grade, the project title, and the student name for that grade. (You will need to JOIN the grades and students table.)

  2. Produce a result set that shows each project, project id, and the number of grades that exist for that project. You will need to JOIN the projects and grades table.

Filtering Using Aggregates

The following queries can be written using either a JOIN or a sub-query.

  1. How many scores for the News Aggregator project were above the average score?

  2. How many scores for the Snake Game were equal to the maximum score?

  3. Which projects have at least 5 grades in the grades table?

Working with Strings

  1. You need to generate text for a grade report that will go out to each

student. Produce a result set for students who received a grade of 90 or above that says "Congrats STUDENT_NAME, you received a SCORE on PROJECT_NAME". You will need to use string concatenation in SQL for this problem.

  1. Produce a similar report as you did in problem #9, but instead, produce a result set for students who received a score of 70 or less. The report should say "Your assignment needs improvement, you received a SCORE on PROJECT_NAME".

  2. There is another database that categorizes students with a student id which is their firstname-lastname. Note that this id is all lowercase with a dash in between. Produce a result set which is the id for the other database for each student. You will need to lower case the first and last name for each student and concatenate strings together to make the full ID.
